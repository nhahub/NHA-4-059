# Git / GitHub workflow

6 people, 10 days — keep this simple enough that it doesn't slow anyone
down, but structured enough that nobody overwrites someone else's work.

## Branching

- `main` — always working. Only updated via PR, after CI passes.
- `develop` — integration branch. Daily merges land here first.
- `feature/<owner-initial>-<short-name>` — e.g. `feature/b-gradcam-vit`,
  `feature/c-logo-insertion`. One branch per task, short-lived (1-2 days max).

## Daily flow

1. Pull `develop`, branch off it: `git checkout -b feature/a-clip-inference develop`
2. Commit small, working increments — don't wait until a whole module is
   "done" to commit. If `download_dataset.py`'s scaffold runs but
   `insert_logo.py` isn't started yet, that's a fine commit.
3. Before opening a PR, run locally:
   ```
   pytest tests/unit tests/integration -v
   ```
4. Open a PR into `develop` (not `main`). CI (`.github/workflows/ci.yml`)
   runs automatically — unit tests, integration tests, config validation,
   Docker build.
5. One other person reviews and approves. With 6 people and 10 days, don't
   block on review for more than a few hours — if no one's free, ping in
   chat and move on to your next task while waiting.
6. Merge `develop` into `main` at the end of each day (Person F owns this),
   after confirming the dashboard still runs against `develop`.

## Commit messages

Keep them factual, not decorative: `Add heatmap energy-ratio scorer` not
`Finally got this working!!`. Reference the owner letter and day if useful:
`[B, Day 3] Add Grad-CAM pass for original test set`.

## What NOT to commit

See `.gitignore` — raw/processed image data, generated heatmaps, MLflow run
logs, and Vercel build artifacts stay out of git. They're regenerated by
running the pipeline, not version-controlled. If the repo starts feeling
slow to clone, check that someone didn't accidentally commit `data/raw/`.

## Day 1 setup (whoever creates the repo)

```bash
git init
git add .
git commit -m "Initial scaffold"
git branch develop
git remote add origin <github-repo-url>
git push -u origin main
git push -u origin develop
```

Then set `develop` as the default branch for PRs in GitHub repo settings,
and enable "Require status checks to pass before merging" for the CI
workflow, so nobody can merge a broken build by accident.
